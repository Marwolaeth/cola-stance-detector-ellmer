# Рекомендации по разрешению референций

## Основной принцип

При анализе позиции автора текста по отношению к целевому объекту используй все доступные контекстуальные сигналы: местоимения, эллипсис, синонимы и подразумеваемые ссылки. Явное называние объекта НЕ требуется.

## Когда разрешать референцию

**Разрешай референцию, если:**

1. Объект явно упомянут в тексте (прямая ссылка)
2. Объект подразумевается через местоимение или эллипсис (пример: "Это опасно" → объект понятен из контекста)
3. Объект логически выводится из темы обсуждения (пример: анализируем отношение к "VPN", текст говорит о "инструментах для обхода блокировок" → это ясная ссылка на VPN)
4. Можно допустить, что объект присутствует в контексте публикации (например, в заголовке, изображении, предыдущих комментариях в цепочке) → **презумпция присутствия**: если нет явных оснований отрицать присутствие объекта, считай его присутствующим

## Когда отказывать в разрешении

**Отказывай разрешать референцию, если:**

1. Предположение о референции требует значительного домысла или фоновых знаний, недоступных из текста
2. Объект противоречит явному содержанию текста

## Примеры

### Пример 1: Разрешение через местоимение

```
Целевой объект: "VPN"
Текст: "Многие используют VPN. [еще 2-3 предложения] Это опасно и неэффективно"
Разрешение: "Это" = VPN (из контекста)
```

### Пример 2: Разрешение через логику

```
Целевой объект: "Западная цивилизация"
Текст: "Депутат брызжет желчью, но его дети живут в Париже"
Разрешение: "брызжет желчью" на Западную цивилизацию (выводится из темы)
```

### Пример 3: Разрешение через синоним

```
Целевой объект: "VPN"
Текст: "Эти инструменты для обхода блокировок опасны"
Разрешение: "инструменты" = VPN (логический вывод)
```

### Пример 4: Презумпция присутствия

```
Целевой объект: "VPN"
Текст: "Это опасно и неэффективно"
Разрешение: "Это" = VPN (нет оснований отрицать присутствие объекта в контексте)
```

### Пример 5: Отказ в разрешении

```
Целевой объект: "VPN"
Текст: "Погода сегодня хорошая"
Разрешение: Нет связи между текстом и объектом
```
